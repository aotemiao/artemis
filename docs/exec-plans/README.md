# Execution Plans

Status: maintained
Last Reviewed: 2026-06-01
Review Cadence: 90 days

这里用于存放复杂任务的执行计划，让计划、决策、风险和进度都成为仓库的一部分。

## 目录约定

- `active/`
  进行中的计划。建议文件名为 `YYYY-MM-DD-<topic>.md`。
- `completed/`
  已完成计划。迁移前应确认结果已落到代码、文档或脚本，风险与遗留问题已记录。
- `templates/`
  计划模板。

## 适用场景

- 跨模块改动
- 需要分阶段推进的工程化工作
- 涉及约束、迁移、重构、基础设施的任务
- 需要把关键决策沉淀给后续 agent / 开发者的任务

## 每个计划至少包含

- 背景
- 目标与非目标
- 范围
- 风险
- 分步任务
- 验证方式
- 决策记录
- 遗留问题

## 与 OpenSpec 的边界

- 执行计划用于记录“怎么做完这项工作”，不替代 OpenSpec 中“规则是什么”的规范变更。
- 如果任务会新增、修改或废弃稳定约束，应在执行计划之外同步更新 OpenSpec artifact。
- 当任务既涉及规范变化，又需要分阶段推进实现时，两条线一起使用：OpenSpec 描述规则变化，执行计划推进实现落地。
