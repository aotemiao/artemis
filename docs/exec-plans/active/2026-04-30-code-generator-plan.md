# Code Generator 专项计划

## 背景

`docs/system-requirements-ddd.md` 中 `GEN-001` 到 `GEN-003` 覆盖代码生成器能力，包括数据库表导入、生成配置维护、预览、下载 ZIP、按路径生成和批量生成。当前仓库尚未建立开发工具上下文，因此需要先明确模块归属、数据模型、模板边界和验证方式。

## 目标

- 建立 `artemis-tool` 或等价开发工具领域服务的分阶段落地路径。
- 将代码生成表、字段配置、数据源元数据读取、模板渲染和 ZIP 输出拆成小闭环。
- 保证生成器只读取元数据和输出文件，不越权修改业务模块源码。

## 非目标

- 不在首个提交中生成所有前后端模板。
- 不直接依赖具体业务模块 infra。
- 不把模板内容散落在 Controller 或命令执行器中。

## 分阶段 Checklist

### GEN-RT-001 工具服务与基础模型

- [ ] 建立开发工具上下文模块，按 client/domain/infra/app/adapter/start 分层。
- [ ] 新增 `gen_tables` 与 `gen_table_columns` 迁移。
- [ ] 建立 `GenTable`、`GenTableColumn` 领域模型和 Gateway。
- [ ] 支持分页查询已导入代码生成列表。
- [ ] 提供 App/Infra/Adapter/ArchUnit 测试和服务 API 文档。

### GEN-RT-002 数据库表导入

- [ ] 支持查询当前可用数据源名称列表。
- [ ] 支持按数据源查询数据库表列表。
- [ ] 支持选择数据库表导入生成配置。
- [ ] 导入时自动读取列名、列描述、数据库类型、是否主键、是否必填等字段元数据。
- [ ] 防止同一数据源下重复导入同一表。

### GEN-RT-003 生成配置维护

- [ ] 支持查看生成表详情。
- [ ] 支持查看生成表字段列表。
- [ ] 支持修改生成配置：包名、模块名、业务名、功能名、作者、模板类型、生成方式、生成路径。
- [ ] 支持字段查询、列表、表单、校验和字典等配置维护。
- [ ] 支持删除生成配置。
- [ ] 支持同步数据库结构到已有生成配置。

### GEN-RT-004 模板渲染与预览

- [ ] 建立模板目录和模板命名规范。
- [ ] 支持后端 DDD/COLA 分层代码预览。
- [ ] 支持 REST DTO、Controller、App 命令/查询、Domain、Infra、Migration、测试骨架预览。
- [ ] 渲染结果按文件路径分组返回，不直接写入源码树。

### GEN-RT-005 代码生成输出

- [ ] 支持下载 ZIP 代码包。
- [ ] 支持按自定义路径生成代码，生成前校验目标路径在允许目录内。
- [ ] 支持批量生成代码。
- [ ] 生成前执行冲突检查，避免覆盖已有源码。
- [ ] 生成后输出文件清单，便于人工 review。

## 验证策略

- 每个阶段至少运行：
  - 新工具模块 scoped Maven verify。
  - `scripts/harness/verify-changed.sh working-tree`。
  - `git -c core.whitespace=blank-at-eol,blank-at-eof,space-before-tab,cr-at-eol diff --check`。
- 模板阶段增加 golden file 测试，确保生成内容稳定。
- 文件输出阶段增加临时目录测试，确认不会写出允许目录。

## 风险与决策

- `2026-04-30`：代码生成器按独立开发工具上下文推进，不塞进 `artemis-system`，避免系统域继续膨胀。
- `2026-04-30`：模板渲染先返回预览和 ZIP，按路径写入能力必须有路径边界校验和冲突检查。
- `2026-04-30`：生成 DDD/COLA 代码时默认遵守当前包结构规范，不生成 app/adapter 直接依赖 infra 的代码。
