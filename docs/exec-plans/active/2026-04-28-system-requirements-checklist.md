# System Requirements 功能补齐 Checklist

## 背景

`docs/system-requirements-ddd.md` 已整理出新系统应覆盖的完整后台管理能力。当前仓库已具备系统域的字典、用户、角色、菜单、内部认证与授权快照，但距离需求文档中的 P0 能力仍缺少租户、组织、岗位、参数、公告、客户端、审计日志、在线用户、资源与工作流等能力。

## 目标

- 将需求缺口拆成可逐项实现、review、验收和提交的小任务。
- 优先补齐 `artemis-system` 中后续功能依赖的基础能力。
- 每个完成项至少包含代码、测试、API/执行计划回写和可解释验证。

## 非目标

- 不在一个提交中一次性实现租户、工作流、资源服务等大块能力。
- 不先补尚未确认范围的演示功能、代码生成器和完整工作流引擎。
- 不改变现有 DDD/COLA 分层和包结构规则。

## 范围

- 优先范围：`artemis-modules/artemis-system`、系统 API 文档、数据库迁移、测试。
- 后续范围：`artemis-auth`、`artemis-resource`、工作流服务或新增领域服务。

## 风险

- 需求文档中大量条目仍标记为“待确认”，实现时先按最小兼容口径落地。
- 多租户和数据权限会影响用户、角色、部门、岗位等后续模型，需要分阶段演进。
- 工作流与资源服务体量较大，应单独建立专项计划。

## 当前实施项

### SYS-002 参数配置基础闭环

验收点：

- [x] 参数 key 唯一。
- [x] 系统内置参数不允许删除。
- [x] 参数可按 key 查询。
- [x] 参数新增、修改、删除和刷新后本地缓存一致。
- [x] 提供 REST 路由文档和 Controller/App/Infra 测试。

### SYS-003 通知公告基础 CRUD

验收点：

- [x] 通知公告支持分页查询和详情查询。
- [x] 通知公告支持创建、修改和逻辑删除。
- [x] 公告包含标题、类型、内容、状态和备注。
- [x] 提供 REST 路由文档和 Controller/App/Infra 测试。

### IAM-003 部门树基础 CRUD

验收点：

- [x] 部门支持平铺列表、树查询、排除指定节点及子节点的树查询、下拉树和详情查询。
- [x] 部门支持创建、修改和逻辑删除，存在子部门时不允许删除。
- [x] 同一父部门下部门名称唯一。
- [x] 新增或移动到父部门时，父部门必须处于正常状态。
- [x] 修改部门父级时同步更新所有子部门的祖级列表。
- [x] 提供 REST 路由文档和 Controller/App/Infra 测试。

### IAM-004 岗位基础 CRUD

验收点：

- [x] 岗位支持分页查询、详情查询、下拉选择和岗位页部门树查询。
- [x] 岗位支持创建、修改和逻辑删除。
- [x] 岗位编码全局唯一。
- [x] 同一部门下岗位名称唯一。
- [x] 已分配给用户的岗位不允许删除。
- [x] 提供 REST 路由文档和 Controller/App/Infra 测试。
- [x] 岗位导出后置到导入导出专项，不阻塞基础 CRUD 闭环。

### IAM-007 客户端授权配置基础 CRUD

验收点：

- [x] 客户端支持分页查询、详情查询、创建、修改、逻辑删除和启停状态维护。
- [x] 客户端包含 `clientId`、`clientKey`、`clientSecret`、授权类型、设备类型、token 活跃超时、token 固定超时、状态和备注。
- [x] 客户端 ID 全局唯一。
- [x] 客户端 key 全局唯一。
- [x] 登录校验链路在用户名密码校验前校验客户端存在、状态正常且支持当前授权类型。
- [x] 提供 REST 路由文档、内部认证契约更新和 Controller/App/Infra/Auth 测试。
- [x] 客户端导出后置到导入导出专项，不阻塞基础 CRUD 闭环。

### AUDIT-002 登录日志写入与查询

验收点：

- [x] 登录日志包含租户、用户账号、客户端、设备类型、IP、地点、浏览器、操作系统、状态、消息和登录时间。
- [x] 认证服务在客户端校验失败、用户名密码失败、登录成功和登出成功时写入登录日志。
- [x] 系统服务支持登录日志分页查询、详情查询、批量逻辑删除和清空。
- [x] 提供 REST 路由文档、Dubbo client 契约更新和 Controller/App/Infra/Auth 测试。
- [x] 登录日志导出和账户解锁后置到导入导出/锁定策略专项，不阻塞当前最小闭环。

### AUTH-004 注册开关、初始密码参数接入

验收点：

- [x] 系统用户创建在未显式传入密码时读取 `sys.user.initPassword`。
- [x] 认证服务提供 `/auth/register`，注册前校验客户端存在、状态正常且支持授权类型。
- [x] 系统服务注册命令读取 `sys.account.registerUser`，关闭时拒绝注册。
- [x] 注册请求校验用户类型，当前最小闭环支持 `SYSTEM` 用户类型。
- [x] 注册成功和失败均写入登录访问日志。
- [x] 提供 Auth API 文档、Dubbo client 契约更新和 App/Auth 测试。

### AUDIT-003 在线用户最小查询与强退入口

验收点：

- [x] 登录成功后写入当前认证实例内的在线用户索引，登出和强退时移除。
- [x] 在线用户包含用户 ID、用户名、token、IP、浏览器、操作系统和登录时间。
- [x] 支持按用户名和 IP 查询在线用户。
- [x] 提供按用户 ID 强退入口。
- [x] 分布式在线会话聚合后置到 Redis/Sa-Token 会话专项，不阻塞当前最小闭环。
