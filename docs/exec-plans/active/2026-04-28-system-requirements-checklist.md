# System Requirements 功能补齐 Checklist

## 背景

`docs/system-requirements-ddd.md` 已整理出新系统应覆盖的完整后台管理能力。当前仓库已具备系统域的字典、用户、角色、菜单、内部认证与授权快照，但距离需求文档中的 P0 能力仍缺少租户、组织、岗位、参数、公告、客户端、审计日志、在线用户、资源与工作流等能力。

## 目标

- 将需求缺口拆成可逐项实现、review、验收和提交的小任务。
- 优先补齐 `artemis-system` 中后续功能依赖的基础能力。
- 每个完成项至少包含代码、测试、API/执行计划回写和可解释验证。

## 非目标

- 不在一个提交中一次性实现租户、工作流、资源服务等大块能力。
- 不先补尚未确认范围的演示功能、代码生成器和完整工作流引擎。
- 不改变现有 DDD/COLA 分层和包结构规则。

## 范围

- 优先范围：`artemis-modules/artemis-system`、系统 API 文档、数据库迁移、测试。
- 后续范围：`artemis-auth`、`artemis-resource`、工作流服务或新增领域服务。

## 风险

- 需求文档中大量条目仍标记为“待确认”，实现时先按最小兼容口径落地。
- 多租户和数据权限会影响用户、角色、部门、岗位等后续模型，需要分阶段演进。
- 工作流与资源服务体量较大，应单独建立专项计划。
