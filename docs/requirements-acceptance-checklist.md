# 需求验收补充清单

> 目标：补齐《Artemis 功能需求梳理》中还不能直接验收的部分。  
> 用法：把本文档作为需求评审表。每个“待确认”项需要产品/业务侧给出明确选择；每个“验收点”应能转成测试用例。  
> 注意：本文档不讨论新系统架构，不指定 DDD 分层、数据库设计、通信协议或技术选型。

关联文档：

- [系统功能需求梳理](./system-requirements-ddd.md)

## 1. 范围决策清单

先把“必须重做”和“可删除/可后置”的功能分开。当前项目里有不少框架演示能力，若不先定范围，需求会被样例功能拖重。

| 编号 | 功能域 | 当前判断 | 建议优先级 | 迁移决策 | 说明 |
| --- | --- | --- | --- | --- | --- |
| SCOPE-001 | 登录、注册、登出、验证码、令牌 | 核心能力 | P0 | 待确认 | 后台系统入口，必须明确登录方式保留范围 |
| SCOPE-002 | 多租户、租户套餐、租户初始化 | 核心能力 | P0 | 待确认 | 若新系统仍是 SaaS/多组织平台，应保留 |
| SCOPE-003 | 用户、部门、岗位、角色、菜单、数据权限 | 核心能力 | P0 | 待确认 | 权限模型是大多数后台功能的基础 |
| SCOPE-004 | 字典、参数、公告 | 基础能力 | P0 | 待确认 | 多数业务模块依赖字典和配置 |
| SCOPE-005 | 操作日志、登录日志、在线用户、账户解锁 | 基础能力 | P0 | 待确认 | 安全审计和运维定位必需 |
| SCOPE-006 | OSS 文件与对象存储配置 | 常用基础能力 | P0/P1 | 待确认 | 如果业务存在附件/头像/导入导出，应保留 |
| SCOPE-007 | 短信、邮件、站内消息 | 集成能力 | P1 | 待确认 | 登录验证码、工作流通知可能依赖 |
| SCOPE-008 | 工作流核心能力 | 业务增强能力 | P0/P1 | 待确认 | 如果新系统需要审批，必须单独细化 |
| SCOPE-009 | 请假申请 | 示例业务 | P2 | 待确认 | 建议仅作为工作流接入样例，不作为核心产品需求 |
| SCOPE-010 | 代码生成器 | 开发工具 | P1/P2 | 待确认 | 对最终业务系统非必需，可后置 |
| SCOPE-011 | SnailJob、Nacos、Admin 控制台入口 | 运维入口 | P1/P2 | 待确认 | 更偏平台运维，不一定是业务需求 |
| SCOPE-012 | Prometheus 服务发现和告警接收 | 运维集成 | P1/P2 | 待确认 | 是否由新架构统一监控决定 |
| SCOPE-013 | Demo 单表、树表、Excel、Redis、ES、MQ、加密、脱敏、国际化示例 | 技术样例 | P2 | 待确认 | 建议从正式需求中剥离，保留为回归样例或删除 |

范围验收标准：

- [ ] 每个功能域标记为“保留 / 删除 / 后置 / 仅样例”。
- [ ] P0 功能均有用例级验收点。
- [ ] 被删除功能明确“不迁移数据、不保留接口、不保留菜单”或给出兼容策略。
- [ ] 被后置功能明确是否保留旧入口、是否影响首版上线。

## 2. 核心用例验收清单

### 2.1 身份认证

#### UC-AUTH-001 获取图形验证码

前置条件：

- 验证码功能开启。

主流程：

1. 用户请求验证码。
2. 系统生成验证码 UUID 和图片。
3. 系统将验证码答案写入缓存。
4. 系统返回验证码开启标记、UUID、图片。

验收点：

- [ ] 返回值包含 `captchaEnabled`、`uuid`、`img`。
- [ ] 同一个 UUID 校验成功或失败后不能重复使用。
- [ ] 验证码关闭时，登录/注册不强制校验图形验证码。

#### UC-AUTH-002 账号密码登录

前置条件：

- 客户端存在且状态正常。
- 客户端授权类型包含 `password`。
- 租户存在、启用、未过期。
- 用户存在且状态正常。

主流程：

1. 用户提交 clientId、grantType、tenantId、用户名、密码和验证码。
2. 系统校验客户端。
3. 系统校验租户。
4. 系统校验验证码。
5. 系统校验用户名和密码。
6. 系统签发访问令牌和刷新令牌。
7. 系统记录登录成功日志。

验收点：

- [ ] 客户端不存在、停用或授权类型不匹配时拒绝登录。
- [ ] 租户为空、不存在、停用、过期时拒绝登录。
- [ ] 验证码过期或错误时拒绝登录并记录失败日志。
- [ ] 密码错误时累计失败次数。
- [ ] 达到最大失败次数后，在锁定时间内拒绝登录。
- [ ] 登录成功后清空失败次数。
- [ ] 返回 access token、refresh token、过期时间、client_id。

#### UC-AUTH-003 短信/邮箱/第三方/小程序登录

验收点：

- [ ] 短信登录校验手机号、短信验证码、租户和客户端授权类型 `sms`。
- [ ] 邮箱登录校验邮箱格式、邮箱验证码、租户和客户端授权类型 `email`。
- [ ] 第三方登录校验来源、授权 code、state、租户和客户端授权类型 `social`。
- [ ] 小程序登录校验 appid、code、租户和客户端授权类型 `xcx`。
- [ ] 第三方账号未绑定时，按产品决策进入“拒绝登录 / 自动注册 / 引导绑定”之一。
- [ ] 第三方账号已绑定但用户不属于当前租户时拒绝登录。

待确认：

- [ ] 新系统首版保留哪些登录方式：`password / sms / email / social / xcx`。
- [ ] 第三方未绑定账号时是否允许自动注册。
- [ ] 短信和邮箱验证码是否必须限流、有效期多久、失败次数如何处理。

#### UC-AUTH-004 用户注册

前置条件：

- 参数 `sys.account.registerUser` 为 `true`。

验收点：

- [ ] 注册开关关闭时拒绝注册。
- [ ] 用户名长度和密码长度符合规则。
- [ ] 用户类型合法。
- [ ] 用户名唯一。
- [ ] 密码加密存储。
- [ ] 注册成功写入登录日志。
- [ ] 注册失败写入失败日志。
- [ ] 多租户开启时注册用户归属指定租户。

待确认：

- [ ] 注册是否允许普通公网用户使用，还是仅管理员创建用户。
- [ ] 注册是否需要默认角色、默认部门、默认岗位。
- [ ] 注册是否计入租户用户额度。

#### UC-AUTH-005 第三方账号绑定和解绑

验收点：

- [ ] 授权地址包含租户 ID、域名、state。
- [ ] 平台来源不支持时返回明确错误。
- [ ] 同一个第三方账号不能绑定多个用户。
- [ ] 同一用户同一平台重复绑定时更新授权信息。
- [ ] 用户可解绑已绑定账号。
- [ ] 解绑失败时返回明确错误。

#### UC-AUTH-006 登出和在线设备

验收点：

- [ ] 登录用户可登出。
- [ ] 登出写入日志。
- [ ] 超级管理员登出时清除动态租户上下文。
- [ ] 未登录调用登出不产生业务错误。
- [ ] 用户可查看自己的在线设备。
- [ ] 用户可强退自己的指定在线设备。

### 2.2 租户

#### UC-TENANT-001 创建租户

前置条件：

- 操作人具有超级管理员角色 `superadmin`。
- 租户套餐存在且启用。
- 企业名称唯一。

主流程：

1. 管理员提交租户信息、套餐、初始用户名和密码。
2. 系统生成唯一 6 位租户编号。
3. 系统创建租户记录。
4. 系统按套餐创建租户管理员角色，角色 key 为 `admin`。
5. 系统创建根部门，名称为企业名称。
6. 系统创建初始系统用户。
7. 系统设置初始用户为根部门负责人。
8. 系统绑定用户与管理员角色。
9. 系统复制默认租户的字典和参数。
10. 系统同步工作流定义到新租户。

验收点：

- [ ] 创建过程任一步失败时整体回滚，不产生半初始化租户。
- [ ] 租户编号唯一。
- [ ] 初始管理员能登录该租户。
- [ ] 初始管理员只能看到套餐授权范围内的菜单。
- [ ] 新租户拥有默认字典、参数和工作流定义。
- [ ] 新租户根部门负责人为初始管理员用户。

待确认：

- [ ] 6 位随机租户编号是否沿用，还是改为业务可读编号。
- [ ] 初始用户的用户名是否允许跨租户重复。
- [ ] 新租户是否必须同步工作流定义。
- [ ] 新租户是否复制 OSS 配置、客户端配置或公告。

#### UC-TENANT-002 修改租户

验收点：

- [ ] 仅超级管理员可修改租户。
- [ ] 可修改联系人、电话、企业信息、域名、有效期、用户额度、状态等。
- [ ] 不允许通过普通修改接口变更租户编号。
- [ ] 不允许通过普通修改接口变更套餐 ID。
- [ ] 停用租户后，该租户用户不能登录。
- [ ] 过期租户不能登录。

#### UC-TENANT-003 删除租户

验收点：

- [ ] 默认管理租户 `000000` 不允许删除。
- [ ] 超级管理员租户记录不允许删除。
- [ ] 删除租户前需要明确是否级联删除租户内用户、部门、角色、业务数据。

待确认：

- [ ] 新系统删除租户采用软删除、禁用归档还是物理删除。
- [ ] 租户已有业务数据时是否允许删除。

#### UC-TENANT-004 租户套餐

验收点：

- [ ] 套餐名称唯一。
- [ ] 套餐可维护菜单集合。
- [ ] 套餐可启用/停用。
- [ ] 已被租户使用的套餐不允许删除。
- [ ] 套餐变更后，可将套餐菜单同步到指定租户。

待确认：

- [ ] 同步套餐时覆盖租户管理员权限，还是只增量补齐。
- [ ] 套餐停用后，已使用该套餐的租户是否受影响。

### 2.3 用户、组织与权限

#### UC-IAM-001 创建用户

验收点：

- [ ] 用户账号、手机号、邮箱唯一。
- [ ] 账号、昵称必填。
- [ ] 用户角色不能为空。
- [ ] 选择的部门在操作者数据权限范围内。
- [ ] 选择的岗位在操作者数据权限范围内。
- [ ] 选择的角色在操作者数据权限范围内。
- [ ] 普通用户不能被分配超级管理员角色。
- [ ] 创建后建立用户-角色、用户-岗位关系。
- [ ] 初始密码来源明确：手动输入或参数 `sys.user.initPassword`。
- [ ] 如果租户有用户额度，创建用户前校验额度。

待确认：

- [ ] 用户名唯一范围是全局唯一还是租户内唯一。
- [ ] 手机号/邮箱唯一范围是全局唯一还是租户内唯一。
- [ ] 用户停用后是否强制踢出在线会话。

#### UC-IAM-002 修改用户

验收点：

- [ ] 不允许操作超级管理员用户。
- [ ] 非超级管理员只能修改数据权限范围内的用户。
- [ ] 修改部门、角色、岗位时重新维护关联关系。
- [ ] 修改失败不能导致用户角色/岗位关系被清空。
- [ ] 重置密码后新密码加密存储。
- [ ] 修改用户状态成功后，新状态影响后续登录。

#### UC-IAM-003 删除用户

验收点：

- [ ] 不允许删除超级管理员用户。
- [ ] 非超级管理员只能删除数据权限范围内用户。
- [ ] 删除用户时同步删除用户-角色关系。
- [ ] 删除用户时同步删除用户-岗位关系。
- [ ] 删除当前登录用户需明确是否允许。

待确认：

- [ ] 用户删除采用软删除还是物理删除。
- [ ] 删除用户后历史日志、流程审批人显示如何处理。

#### UC-IAM-004 部门树维护

验收点：

- [ ] 同一父部门下部门名称唯一。
- [ ] 停用父部门下不允许新增子部门。
- [ ] 修改父部门时同步更新子孙部门祖级路径。
- [ ] 启用子部门时自动启用全部上级部门。
- [ ] 查询、修改、删除部门时校验数据权限。
- [ ] 删除部门前是否检查子部门、岗位、用户，需要产品确认。

#### UC-IAM-005 岗位维护

验收点：

- [ ] 同一部门下岗位名称唯一。
- [ ] 岗位编码唯一。
- [ ] 已分配用户的岗位不允许删除。
- [ ] 岗位可启用/停用。

#### UC-IAM-006 角色维护

验收点：

- [ ] 角色名称唯一。
- [ ] 角色权限字符串唯一。
- [ ] 不允许操作超级管理员角色。
- [ ] 不允许新增内置管理员角色 key：`superadmin`、`admin`。
- [ ] 不允许修改内置管理员角色 key。
- [ ] 已分配用户的角色不允许删除。
- [ ] 已分配用户的角色不允许禁用。
- [ ] 修改角色菜单权限后，应使在线用户权限及时生效。
- [ ] 修改角色数据权限后，应清理角色数据权限缓存。
- [ ] 不允许修改当前登录用户自己的角色授权。

#### UC-IAM-007 菜单与权限

验收点：

- [ ] 菜单支持目录、菜单、按钮三类。
- [ ] 同一父级下菜单名称唯一。
- [ ] 目录/菜单的路由配置唯一。
- [ ] 角色菜单树能回显已授权菜单。
- [ ] 租户套餐菜单树能回显套餐授权菜单。
- [ ] 批量级联删除菜单时同步删除角色菜单关系。
- [ ] 超级管理员和租户管理员可获取完整授权范围内的菜单列表。

待确认：

- [ ] 新系统是否仍由后端生成前端路由。
- [ ] 权限码是否完全兼容旧系统。

#### UC-IAM-008 客户端授权配置

验收点：

- [ ] 客户端可增删改查、导出、启停。
- [ ] clientId 唯一。
- [ ] 登录时必须校验客户端状态。
- [ ] 登录时必须校验 grantType 属于客户端授权范围。
- [ ] token 活跃超时和固定超时按客户端生效。

### 2.4 系统基础数据

#### UC-SYS-001 字典

验收点：

- [ ] 字典类型编码唯一。
- [ ] 字典类型下存在字典数据时不允许删除。
- [ ] 修改字典类型编码时同步更新字典数据。
- [ ] 字典缓存可刷新。
- [ ] 按字典类型查询字典数据时能命中缓存。

#### UC-SYS-002 参数

验收点：

- [ ] 参数 key 唯一。
- [ ] 系统内置参数不允许删除。
- [ ] 参数可按 key 查询。
- [ ] 参数新增、修改、删除后缓存一致。
- [ ] 注册开关、初始密码、OSS 预览开关等关键参数有明确默认值。

#### UC-SYS-003 通知公告

验收点：

- [ ] 公告可增删改查。
- [ ] 公告类型支持通知、公告。
- [ ] 公告状态支持正常、关闭。

### 2.5 审计与监控

#### UC-AUDIT-001 操作日志

验收点：

- [ ] 新增、修改、删除、导入、导出、授权、强退、清空等操作写入操作日志。
- [ ] 操作日志包含操作人、模块、业务类型、URL、请求方式、IP、请求参数、返回结果、状态、耗时。
- [ ] 操作日志可查询、导出、删除、清空。
- [ ] 操作日志清空需单独权限控制。

#### UC-AUDIT-002 登录日志

验收点：

- [ ] 登录成功、登录失败、登出、注册写入登录日志。
- [ ] 登录日志包含租户、账号、客户端、设备、IP、浏览器、系统、状态、消息。
- [ ] 登录日志可查询、导出、删除、清空。
- [ ] 账户解锁可清除失败次数或锁定状态。

#### UC-AUDIT-003 在线用户

验收点：

- [ ] 管理员可按用户名/IP 查询在线用户。
- [ ] 管理员可强退指定在线用户。
- [ ] 用户可查询和强退自己的在线设备。
- [ ] 角色权限变化后，可按角色或用户清理在线会话。

### 2.6 资源与消息

#### UC-RES-001 文件上传下载

验收点：

- [ ] 上传文件不能为空。
- [ ] 上传成功写入文件记录。
- [ ] 上传成功返回文件 ID、URL、原始文件名等信息。
- [ ] 下载不存在的文件返回业务错误。
- [ ] 删除文件时删除文件记录，并尝试删除对象存储对象。
- [ ] 按 ID 列表查询文件时保持多文件返回能力。

#### UC-RES-002 OSS 配置

验收点：

- [ ] 配置 key 唯一。
- [ ] 内置配置不允许删除。
- [ ] 可切换启用/默认配置。
- [ ] 配置包含 bucket、endpoint、region、访问策略、HTTPS、自定义域名。

待确认：

- [ ] 新系统保留哪些存储商：MinIO、阿里云、腾讯云、七牛或所有 S3 兼容服务。
- [ ] secretKey 是否需要加密存储。

#### UC-RES-003 消息

验收点：

- [ ] 支持向指定用户发送站内消息。
- [ ] 支持全员站内消息。
- [ ] 支持短信单发、群发、模板、异步、延迟、黑名单。
- [ ] 支持邮件发送。
- [ ] 工作流可按站内信、邮箱、短信发送通知。

待确认：

- [ ] 是否需要消息发送记录和失败重试。

这些是当前仍缺的“业务侧答案”，不属于架构设计：

1. 保留范围：确认第 1 节每个功能域是保留、删除、后置还是仅样例。
2. 默认角色：确认新系统首版内置哪些角色，以及每个角色默认菜单和按钮权限。
3. 用户唯一性范围：账号、手机号、邮箱是全局唯一还是租户内唯一。
4. 用户注册策略：是否开放注册，注册后归属部门/角色如何确定。
5. 租户删除策略：禁用、软删除、物理删除、是否允许有业务数据时删除。
6. 租户套餐同步策略：覆盖、增量、还是生成差异审批。
7. 文件存储策略：保留哪些存储商，旧文件 URL 是否兼容。
8. 消息策略：哪些业务事件发站内信、短信、邮件，失败是否重试。
9. 工作流保留范围：是否保留流程引擎全部能力，是否迁移运行中流程。
10. 请假示例定位：正式业务、样例业务或删除。
11. 日志保留策略：操作日志和登录日志迁移范围、保留周期、清理权限。
12. 代码生成器定位：首版保留、后置或删除。
13. 运维控制台定位：作为菜单入口保留，还是由新运维平台接管。

## 9. 建议的需求评审顺序

1. 先评审第 1 节范围决策，砍掉不迁移的示例功能。
2. 再评审第 8 节待补需求输入，给出业务侧答案。
3. 然后把第 2 节 P0 用例转成验收测试用例。
4. 最后按第 6 节做数据迁移专项评审，尤其是工作流运行数据。

完成标准：

- [ ] P0 功能没有“待确认”。
- [ ] 每个 P0 用例都有可执行验收测试。
- [ ] 每条业务规则都有正向和反向测试。
- [ ] 权限矩阵能映射到菜单权限码和数据权限。
- [ ] 数据迁移范围和兼容策略已签字确认。
