# Security Review Checklist

Status: maintained
Last Reviewed: 2026-06-02
Review Cadence: 90 days

本清单用于审查 agent 或开发者提交的高风险改动，重点覆盖权限、幂等、锁、事务、异常处理、SQL 性能、日志和可观测性。它不是替代代码 review，而是让 reviewer 不遗漏固定风险面。

## 适用场景

以下任一条件成立时，最终 handoff 必须引用本清单或说明不适用原因：

- 涉及登录、鉴权、角色、权限点、数据权限、内部接口或用户上下文透传。
- 涉及写操作、批量操作、删除、状态机、重复提交或外部回调。
- 涉及跨服务调用、消息、异步任务、缓存、锁或事务边界。
- 涉及新增 SQL、分页查询、导出、报表或大范围列表。
- 涉及敏感数据、审计日志、错误码、可观测性或运维排障。

## 审查清单

| 类别 | 检查问题 | 合格信号 |
|------|----------|----------|
| 权限 | 是否校验登录态、角色、权限点、数据权限和内部接口访问边界 | Controller / Gateway / App 层有清晰权限入口，测试覆盖无权限和越权 |
| 幂等 | 重复请求、重复导入、重复回调是否会产生重复数据或重复副作用 | 唯一约束、幂等键、状态机或重复检测已落地 |
| 并发 / 锁 | 并发创建、并发替换、并发状态流转是否会竞态 | 数据库约束、乐观锁、分布式锁或重试策略明确 |
| 事务 | 写操作事务边界是否在 app 层用例处，失败是否回滚 | `@Transactional` 位于用例编排边界，领域规则不依赖 infra 细节 |
| 一致性 | 跨服务、缓存、会话、审计是否存在最终一致性窗口 | 同步 / 异步策略、补偿或缓存失效说明清楚 |
| 异常处理 | 错误码、响应格式、日志级别是否稳定 | 失败路径测试覆盖，错误消息不泄露敏感数据 |
| SQL 性能 | 分页、排序、过滤、导出是否有索引和查询范围 | 查询有边界，避免 N+1 和无条件全表扫描 |
| 日志 / 审计 | 关键操作是否可追踪、可审计、可排障 | traceId、用户 ID、业务 ID、审计事件或状态日志可定位 |
| 可观测性 | smoke、readiness、状态 API 或日志入口是否能证明行为 | 验证映射引用真实脚本、端点或人工验收步骤 |

## Handoff 要求

agent 完成高风险任务时，最终说明至少包含：

- 哪些清单项适用。
- 每个适用项对应的代码、测试或人工检查入口。
- 哪些风险未自动化验证，以及为什么不能自动化。
